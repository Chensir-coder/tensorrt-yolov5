> 针对工业级目标检测场景中 PyTorch 框架推理延迟高、显存占用大的痛点，独立完成了 YOLOv5s 从算法模型到硬件部署的全链路优化。实现了 PyTorch → ONNX → TensorRT 的标准转化流程，深入底层算子融合与显存复用机制。重点攻克了 **INT8 训练后量化（PTQ）** 中的精度回迁难题，通过手写自定义校准器（Calibrator）并采用熵校准（Entropy）算法，在 COCO 数据集上实现了精度无损（mAP 掉点 < 0.8%）前提下的 **4.2 倍加速**（从 18ms 降至 4.3ms）。同时封装了高内聚的 Python 推理 SDK，支持单图与批量推理，具备极强的工程落地属性。

**核心技术栈：** PyTorch、ONNX、TensorRT、CUDA、Python、INT8 Quantization、NMS 优化。

### 4.4 INT8 校准器核心逻辑（面试必问）

```python
import tensorrt as trt
import pycuda.driver as cuda
import numpy as np

class YOLOv5Calibrator(trt.IInt8Calibrator):
    def __init__(self, image_paths, batch_size=32):
        self.image_paths = image_paths
        self.batch_size = batch_size
        self.current_idx = 0
        # 预加载所有图片并预处理为模型输入格式 (NCHW)
        self.buffers = self._load_images()
        
    def get_batch(self, names):
        if self.current_idx >= len(self.buffers):
            return None
        batch = self.buffers[self.current_idx:self.current_idx + self.batch_size]
        self.current_idx += self.batch_size
        # 返回 GPU 显存地址指针
        return [int(cuda.mem_alloc(batch.nbytes))]  # 简化版，实际需用 cuda.memcpy_htod
        
    def get_algorithm(self):
        return trt.CalibrationAlgoType.ENTROPY_CALIBRATION_2  # KL 散度校准
```

> 💡 **关键理解（面试话术）：** 校准器的作用是通过 KL 散度找到最优截断阈值 T，使得 FP32 分布映射到 INT8 后信息损失最小。校准集需要覆盖白天、夜晚、大目标、小目标等多样场景。

## 五、关键点（面试必背）

### 5.1 TensorRT 优化三把斧

| 优化手段 | 原理 | 效果 |
|----------|------|------|
| **算子融合** | 消除多余内存读写，如 Conv + BN + ReLU → CBR 单核 | 减少 Kernel Launch 次数和显存访问 |
| **内核自动调优** | 针对矩阵形状选择最优 CUDA Kernel（不同 Tile Size） | 充分利用 Tensor Core |
| **显存复用** | 构建时分析张量生命周期，用一块显存池服务多个中间变量 | 降低显存峰值占用 |

### 5.2 INT8 量化精髓（面试手撕题）

**量化 = 映射（Mapping）+ 截断（Clipping）**

- FP32 范围：约 [-2³¹, 2³¹]
- INT8 范围：[-127, 127]

校准器通过 **KL 散度（相对熵）** 找到最优截断阈值 T，使得量化前后分布差异最小。

> ❓ **为什么需要校准？** 因为 FP16 是 IEEE 标准直接支持，而 INT8 需要根据数据分布确定映射参数。

### 5.3 后处理延迟陷阱

模型推理 3ms，但 CPU NMS 可能花 5ms。

> ✅ **解药：** 使用 `cudaMemcpyAsync` 异步传回 CPU，配合 CUDA Stream，让拷贝与后续计算重叠，CPU 侧用向量化 Numpy 加速。

## 十、面试高频追问 & 标准答案

### Q1：FP16 和 INT8 的加速原理分别是什么？

FP16 利用 Tensor Core 硬件支持，半精度数据带宽减半、计算单元翻倍；INT8 在 FP16 基础上进一步压缩，依赖校准器确定最优截断阈值。

### Q2：校准集选多少张？为什么？

500-1000 张。过少无法代表数据分布，过多校准时间长且收益递减。**覆盖多种场景（光照、尺度、类别）比数量更重要。**

### Q3：如果 INT8 掉点超过 1%，你会怎么调？

1. 检查校准集是否与训练集分布一致；
2. 改用 `ENTROPY_CALIBRATION_2` 算法；
3. 对敏感层（如检测头）跳过量化（`--precisionConstraints=prefer`）。

### Q4：TensorRT 的 Engine 文件是跨 GPU 通用的吗？

**不通用。** Engine 与 GPU 架构（SM 版本）、CUDA 版本、TensorRT 版本强绑定。3090（Ampere）生成的 Engine 无法在 V100（Volta）上运行。
